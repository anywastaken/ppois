# PPOIS1: Система управления личными финансами (CLI)

## Индивидуальный вариант и предметная область
Реализована программная система для предметной области: **учет личных финансов, бюджетирование и управление инвестициями.**

**Ключевые сущности:**
- **Банковский счет:** управление балансом пользователя.
- **Бюджетные категории:** контроль лимитов трат по категориям.
- **Инвестиционный портфель:** инструменты для вложения средств под процент.
- **Транзакции:** история всех финансовых перемещений.
- **База данных:** персистентное хранение в формате JSON.

**Поддерживаемые операции:**
- Создание и удаление банковских счетов.
- Депозит, снятие и межбанковские переводы.
- Установка лимитов по категориям трат (бюджет).
- Создание инвестиций с расчетом доходности.
- Переводы между счетами и инвестиционными инструментами.
- Автоматическое сохранение состояния в файлы.

---

## Объектно-ориентированный анализ

### Сущности
- `Transaction`: модель единичной операции (отправитель, получатель, сумма, уникальный UUID).
- `Budget`: категория расходов с установленным лимитом.
- `Investment`: инструмент вложения средств с процентной ставкой.
- `BankAccount`: основной агрегат, оркестрирующий логику всех модулей.
- `Databases` (`BudgetDatabase`, `InvestmentDatabase`, `TransactionHistory`): слои DAO (Data Access Object) для работы с файлами.



### Варианты использования (Use Cases)
1. **Совершение покупки:** система списывает средства со счета и автоматически проверяет остаток лимита в соответствующем бюджете.
2. **Управление инвестициями:** пользователь создает инструмент, после чего может переводить средства с расчетных счетов в инвестиции.
3. **Инициализация системы:** при запуске проверяется целостность JSON-файлов в директории `Database/`, при необходимости создаются новые.

### Разделение ответственности (SoC)
- **Domain Models**: `Budget.py`, `Investment.py`, `Transaction.py` — чистые модели данных.
- **Data Access Layer**: Классы с постфиксом `Database` инкапсулируют логику работы с `json` и `pathlib`.
- **Service Layer**: `BankAccount.py` содержит бизнес-правила (проверка баланса, логика транзакций).
- **Presentation Layer**: `Menu.py` и `main.py` отвечают за взаимодействие с пользователем (CLI).

## Структура проекта
```text
main.py                # Точка входа в приложение
BankAccount.py         # Основной контроллер бизнес-логики
Budget.py              # Модель бюджета
BudgetDatabase.py      # Хранилище и логика бюджетов
Investment.py          # Модель инвестиции
InvestmentDatabase.py  # Хранилище инвестиций
Transaction.py         # Модель транзакции (UUID)
TransactionHistory.py  # Хранилище и экспорт истории
Exception.py           # Кастомные исключения (IncorrectDataInFileException)
Menu.py                # Логика консольного меню
Database/              # Директория с JSON-файлами (создается автоматически)
tests/                 # Набор Unit-тестов (Pytest)
```
## Запуск CLI
```bash
# Активация окружения (Linux)
source .venv/bin/activate

# Запуск
python3 main.py
```
