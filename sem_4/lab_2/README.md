# Лабораторная работа №2 (Вариант №6)

## Назначение
Приложение для учета товаров на складе (Вариант №6) с графическим интерфейсом. Реализует функции добавления, поиска и удаления записей, отображение данных в таблице с постраничной навигацией (по 10 записей), а также импорт и экспорт данных в формате XML.

## Технологии
- **Python 3.10+**
- **PyQt6** (Интерфейс пользователя)
- **XML**: `xml.sax` (SAX-парсер для чтения), `xml.dom.minidom` (DOM-парсер для записи)
- **Архитектура**: MVC (Model-View-Controller)

## XML-парсеры
- **SAX-парсер**: Класс `ProductHandler` в `model/Database.py`. Обеспечивает быстрое чтение XML-файла и создание объектов `Product`.
- **DOM-парсер**: Используется в методе `save_to_xml` для генерации структурированного XML из текущего списка товаров.

## Структура и классы

| Файл | Класс / Функция | Назначение |
| :--- | :--- | :--- |
| `main.py` | `main()` | Точка входа: создание `QApplication`, `MainWindow`, `Database` и `Controller`. |
| `controller/Controller.py` | `Controller` | Обработка сигналов UI, управление пагинацией и связь между Моделью и Представлением. |
| `model/Product.py` | `Product` | Модель данных товара (Название, Производитель, УНП, Кол-во, Адрес). |
| `model/Database.py` | `Database` | Логика работы с данными: CRUD-операции, фильтрация и XML-парсинг. |
| `view/MainWindow.py` | `MainWindow` | Главное окно приложения, таблица товаров и кнопки управления страницами. |
| `view/AddProductDialog.py` | `AddProductDialog` | Диалог для ввода данных при добавлении нового товара. |
| `view/SearchProductDialog.py` | `SearchProductDialog` | Окно поиска с собственной постраничной навигацией результатов. |
| `view/DeleteProductDialog.py` | `DeleteProductDialog` | Окно для удаления группы товаров по заданным фильтрам. |

## Данные 
- **Атрибуты товара**: Название, Производитель, УНП (9 цифр), Количество, Адрес склада.
- **Файлы данных**: 
    - `data.xml` — основная база на 50 записей.
    - `data_2.xml` — второй файл на 50 записей для проверки импорта.
